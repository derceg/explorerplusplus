Edit/Paste

This menu item takes data from the clipboard and pastes it into the files pane, even if the folders pane has the focus.  If files or folders were cut in order to place them on the clipboard, then they are deleted at this time.  Explorer++ handles various types of data, and the pasting method, differently, according to the following table:

Clipboard data

Paste method

Result
text paste into dialog, or rename file operation text inserted, or overwrites text selection
text paste into blank area of files pane text pasted as new text file, automatic naming:
eg. Clipboard Text (10-20-2010, 6.44.01 PM).txt
text paste into folders pane (using menu item, or toolbar button) text pasted as new text file (as above) into Files pane (current active folder), even if the folders pane has the focus.
bitmap
(image)		 image data, copied from a paint program, for example is pasted into blank area of files pane new image created, automatic naming:
eg. Clipboard Image (10-20-2010, 6.53.48 PM).bmp
New image always created as Windows bitmap, regardless of original image type.

See Copy for a discussion of behavior when file names are duplicated.
